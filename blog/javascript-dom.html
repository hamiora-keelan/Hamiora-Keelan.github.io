Why can Javascript do all that?

Imagine HTML as the car's frame and engine block. Then comes CSS, the car's paint job and upholstery. It determines how the web page looks, from the colours and fonts to the layout and styling, making it visually appealing and comfortable to use.

  • letdetermines the code to be a variable. Old school JavaScript used to use var

  • "value"You put data (the value) into the variable, like putting something in the box. This data can be numbers, text, lists, or even more complex values.

Objects and Arrays

Objects and arrays provide the clear organisation of information, to later be accessed by functions, loops and more.

Objects

Let's use the conference analogy again, but this time, let's imagine the hat doesn't just represent the speakers name. The hat also represents the speakers allotted presentation duration, the presentation topic and a contact email for following up. In JS this may look something like this:

const greenHat = {name: 'Mike', duration: 5, topic: 'cleaning', email: 'mike@cleanco.com'}

const determines the code to be an object.

Array

Objects serve as valuable tools for organizing a comprehensive set of information related to a specific topic, particularly when that topic entails multiple aspects. However, in scenarios requiring a more straightforward structure, arrays may be more appropriate. Arrays excel at storing data sequentially and orderly. Consider a situation where Mike wishes to identify the individuals seated at his table during a conference. Mike finds the seating arrangement (Josh, Jesse, Maddy, Fred, Mike, Sarah) , each seat is labeled from [0] to [5] representing the six seats on the table. Mike is seated on seat [4] and he now wants to know who he is seated next to. How can he do this using an Array? const seatingArrangement = ["Josh", "Jesse", "Maddy", "Fred", "Mike", "Sarah"];

Functions

In the world of JavaScript, functions play a crucial role akin to the trusty knife and fork you use during a meal. Just as these utensils are essential for cutting and eating your food, functions are vital tools for organizing and executing your code.

Picture yourself at a dinner table, faced with a hearty meal. As you pick up your knife and fork, you instinctively know how to use them to cut your food into manageable pieces and bring them to your mouth for consumption. Similarly, functions in JavaScript are designed to perform specific tasks or operations within your code.

Think of functions as modular units of code that you can reuse throughout your script. Just as you can use the same knife and fork to eat different dishes, you can employ functions multiple times in your code to execute the same task wherever it's needed. This reusability not only saves time but also promotes efficiency and consistency in your programming endeavors.

When you invoke a function, you're essentially delegating a specific task to it, much like how you rely on your knife and fork to handle different aspects of your meal. You don't need to worry about the inner workings of the function; you simply provide it with the necessary input and expect the desired output.

Control Flow

I am fortunate enough to travel more frequently than most, but I am not fortunate enough to have the privilege of priority boarding. So, I get to sit there, and observe each passenger boarding in order of priority, starting with the old and sickly, progressing to premium and loyalty-based passengers, and finally, to economy passengers. A simple sequence, however, not all airlines board planes with the same sequence. Javascript control flow refers to the order that code is executed. The default order of execution is top to bottom, but like an airline, you are able to modify the order of which the code is executed using statements.

Conditonal Statements

Conditonal statements execute code on the fufilment of certain conditons, the most common conditions are:

  • if executes the code providing the condition is true.
  • else if provides an additonal condition if if returns false.
  • else executes if all remaining conditions return false.
let age = 18; if (age >= 18) { console.log("You are eligible to vote."); } else { console.log("You are not eligible to vote."); }

Looping Statements

Looping enables the repetitive execution of code, some examples are:

  • for uses a counter variable to execute code repeatedly.
  • while providing a condition is true, the code will be repeatedly executed.
    • // Print numbers from 1 to 5 for (let i = 1; i <= 5; i++) { console.log(i); }

Understanding control flow and mastering the manipulation of execution order through statements empower you to create sophisticated programs capable of making decisions based on diverse parameters.

The peoples favorite.... Loops

Expanding on loop statements, loops can repeatedly complete a defined task and activate on certain conditons. For example, Mike works for a cleaning company, Mike knows he has three jobs on Tuesday, one in Te aro, one in Johnsonville and one in Newlands, Mike also knows he needs to alert his boss when the jpb is complete. What could this look like as code?

const locations = ["Te aro", "Johnsonville", "Newlands"];for (let i = 0; i < locations.length; i++) { const currentLocation = locations[i]; console.log(`Cleaning at ${currentLocation}`); console.log(`Job completed at ${currentLocation}, notifying boss.`);

The const locations array holds the names of the three suburbs where Mike has jobs. The for (let i = 0; i < locations.length; i++) loop iterates through the locations array. Inside the loop: The current location is retrieved using its index (i) then gets stored in currentLocation. A message is logged simulating cleaning the job at the current location. Another message is logged simulating alerting the boss that the job is complete.

Summary

JavaScript's magic comes from its ability to manipulate web pages. It works alongside HTML (structure) and CSS (styling) to bring web pages to life. Imagine HTML as the bones of a web page, CSS as the muscles and skin, and JavaScript as the brain. JavaScript interacts with the Document Object Model (DOM), a blueprint of a web page. By modifying the DOM, JavaScript can dynamically change content, add interactivity, and respond to user actions. Core building blocks like variables, functions, and control flow statements allow JavaScript to organize data, perform tasks efficiently, and make decisions based on conditions. This combination enables complex and engaging web experiences.
